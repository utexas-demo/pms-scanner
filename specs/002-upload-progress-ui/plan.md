# Implementation Plan: Upload Progress Dashboard

**Branch**: `002-upload-progress-ui` | **Date**: 2026-04-08 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/002-upload-progress-ui/spec.md`

---

## Summary

Add a browser-accessible real-time upload progress dashboard to the existing pms-scanner service. A new `StatusStore` singleton (thread-safe in-memory) is shared between the scanner worker thread and a FastAPI/uvicorn HTTP server running in a background thread. The scanner worker publishes status transitions (`pending → uploading → success | failed`) to the store; the FastAPI app exposes Server-Sent Events so connected browsers update within 1 second of each transition. The dashboard is a single HTML page served from `GET /` with no build step.

## Module Annotations

### `scanner/store.py` (NEW)

Core of this feature. Contains:

- `Status` — `enum.Enum` with values `PENDING`, `UPLOADING`, `SUCCESS`, `FAILED` (serialises as lowercase strings for JSON/SSE)
- `FileRecord` — `@dataclass` with fields: `id: str`, `filename: str`, `status: Status`, `detected_at: datetime`, `updated_at: datetime`, `error_message: str | None`, `attempts: int`; `.to_json()` returns a JSON string for SSE payloads
- `StatusStore` — class with:
  - `_records: dict[str, FileRecord]`
  - `_lock: threading.Lock`
  - `_subscribers: list[asyncio.Queue[str]]`
  - `_loop: asyncio.AbstractEventLoop | None`
  - `add(record)`, `update(id, *, status, error_message, attempts)`, `all()`, `subscribe()`, `unsubscribe(q)`, `set_loop(loop)`, `_broadcast(payload)`
- `status_store: StatusStore` — module-level singleton (imported by `watcher.py` and `api.py`)

### `scanner/api.py` (NEW)

- `DASHBOARD_HTML: str` — complete dashboard HTML page as a string constant; uses vanilla JS `EventSource` to connect to `/api/events` and update a table of `FileRecord` rows
- `app: FastAPI` — application instance
- `GET /` → `HTMLResponse(DASHBOARD_HTML)`
- `GET /health` → `{"status": "ok"}`
- `GET /api/files` → `list[dict]` from `status_store.all()`
- `GET /api/events` → `EventSourceResponse` (sse-starlette); subscribes to `status_store`, yields events, unsubscribes on disconnect
- `@app.on_event("startup")` → `status_store.set_loop(asyncio.get_event_loop())`

### `scanner/watcher.py` (updated)

Add three `status_store` calls:
1. `ImageEventHandler._handle()` after `_queue.put(path)`: `status_store.add(FileRecord(id=uuid4_str, filename=path.name, status=Status.PENDING, ...))`; store the `id` in the queue item alongside the `Path` — change queue item type from `Path` to a named tuple or dataclass `QueueItem(path: Path, record_id: str)`
2. `process_file()` before `upload_image()`: `status_store.update(record_id, status=Status.UPLOADING)`
3. `process_file()` after `upload_image()` returns: `status_store.update(record_id, status=Status.SUCCESS|FAILED, error_message=..., attempts=result.attempts)`

### `scanner/config.py` (updated)

Add one field:
```python
dashboard_port: int = 8080
```

### `scanner/__main__.py` (updated)

Start uvicorn server in a background thread before the watchdog observer:
```python
import uvicorn, threading
from scanner.api import app

config = uvicorn.Config(app, host="0.0.0.0", port=settings.dashboard_port, log_level="warning")
server = uvicorn.Server(config)
api_thread = threading.Thread(target=server.run, name="api-server", daemon=True)
api_thread.start()
```
Shutdown: `server.should_exit = True` before `observer.stop()`.

## Key Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| SSE library | `sse-starlette` | Production-ready, Python 3.12 compatible, handles disconnects |
| ASGI server | `uvicorn[standard]` daemon thread | Lifecycle control; daemon exit means clean shutdown |
| Thread→asyncio bridge | `asyncio.run_coroutine_threadsafe` | Idiomatic; returns Future for error propagation |
| State persistence | In-memory only | Spec FR-005; no database for v1 |
| HTML serving | Embedded string constant | No build step; single page; trivial to maintain |
| Queue item type | `QueueItem` named dataclass | Carry record_id alongside path for store updates |
